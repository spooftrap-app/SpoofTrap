## 2024-05-24 - Command Injection Vulnerability via String Interpolation in Process Arguments
**Vulnerability:** A command injection vulnerability was found in `Sources/UpdateChecker.swift` where `appPath` (the path to the application bundle) was passed to `/bin/sh -c` via string interpolation (`task.arguments = ["-c", "sleep 1 && open \"\(appPath)\""]`). If an attacker could control or modify the path to the application to include shell metacharacters, it could lead to arbitrary command execution during the update process.
**Learning:** String interpolation in shell commands, even when seemingly protected by quotes, is susceptible to command injection if the interpolated variable can be manipulated.
**Prevention:** When executing shell commands via `Process()`, always pass dynamic or user-controlled variables as separate arguments using parameter substitution (e.g., passing `$0` in the `-c` script and appending the variable to the arguments array). This ensures the variable is treated strictly as an argument by the shell and not executed as part of the command string.
