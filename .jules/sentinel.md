## 2026-04-09 - Found Hardcoded Supabase Anon Key\n**Vulnerability:** A hardcoded Supabase anonymous key was found in `Sources/ProFeatures/LicenseManager.swift`. While anon keys are designed to be public to an extent when used with Row Level Security (RLS), it's still a risk to hardcode them in open-source or widely distributed repositories.\n**Learning:** Hardcoded credentials can easily slip into codebases if not using proper environment variable management or configuration structures.\n**Prevention:** Move keys to a `Secrets.swift` file and retrieve them via property list, environment variables, or other secure configurations. If it is intended to be bundled, we must acknowledge it is technically public but should ideally not reside plainly as a string in the code if we want to avoid scraping.
## 2026-04-09 - Found Command Injection Vulnerabilities\n**Vulnerability:** Several places execute system commands unsafely via `Process()`. Specifically, `/sbin/ping` takes `ip` directly as an argument, and `robloxAppPath` is passed directly into `open -n -a`. If `ip` or `robloxAppPath` can be influenced by the user, this allows arbitrary command execution or options injection. Though `ip` comes from Roblox logs, logs can be poisoned if user-controlled text is logged. `robloxAppPath` is directly user-controllable.\n**Learning:** Swift's `Process()` is safer than `system()`, but passing user-controlled strings as arguments without validation can still lead to options injection or command injection depending on the executable being called.\n**Prevention:** Validate input strings to ensure they contain only expected characters (e.g. valid IP addresses or file paths) and do not begin with hyphens if passed as positional arguments.
## 2026-04-09 - Apparent Insecure HTTP Connection\n**Vulnerability:** In `Sources/RobloxLogWatcher.swift`, the application makes an HTTP GET request to `http://ip-api.com/json/\(ip)`. Unencrypted traffic is susceptible to man-in-the-middle attacks where an attacker could spoof the response.\n**Learning:** Hardcoding `http://` URLs for API calls should be avoided when `https://` is available, as it exposes data in transit and opens up potential tampering.\n**Prevention:** Use `https://` for all external API calls.
